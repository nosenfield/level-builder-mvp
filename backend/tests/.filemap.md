# backend/tests File Map

Integration and unit tests for the backend library. Tests are separated from source files for clarity and to reduce source file size.

## Files

### validation_tests.rs
Unit tests for Space JSON validation logic.

**Tests:**
- Schema version validation (accepts v1, rejects others)
- Block count limits (accepts 0-10000, rejects >10000)
- Coordinate bounds (X/Z: -1000 to 1000, Y: 0 to 1000)
- Color format validation (`#RRGGBB` and `#RGB` formats)
- Duplicate position detection
- Full `validate_space_json()` integration

### rbxlx_tests.rs
Unit and integration tests for RBXLX file generation.

**Tests:**
- `hex_to_color3()` color conversion (formats, case insensitivity, invalid input)
- `calculate_spawn_position()` spawn calculation (empty, single, multiple blocks)
- `generate_rbxlx()` file generation (empty, single, multiple, max blocks)
- Color palette coverage (all 10 editor colors)
- Coordinate edge cases (boundary values)
- Baseplate creation and properties
- SpawnLocation fixed positioning and Decal child

## Running Tests

```bash
cargo test                    # Run all tests
cargo test validation         # Run validation tests only
cargo test rbxlx              # Run rbxlx tests only
cargo test -- --nocapture     # Show println! output
```
